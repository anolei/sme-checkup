# 优质中小企业自检助手（开源版）

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](https://opensource.org/licenses/MIT)
[![Version](https://img.shields.io/badge/version-v1.0--open-green)]()

> 面向中小企业成长培育的专业化自检 Skill。  
> 帮助企业围绕经营质量、创新能力、知识产权、产业链定位和核心技术叙事开展系统化自检，尽早发现短板，减少信息不对称。

---

## 为什么开源

这个 Skill 凝结了我们多年服务中小企业的工作经验和工作实践。

选择开源，是经过深思熟虑后的决定。我们希望把这项能力作为送给广大中小企业的一份礼物，帮助企业减少信息不对称，更早发现问题、更准识别短板；同时，也希望为 AI 开源社区贡献一个真正来自专业服务一线、可复用、可共创的领域 Skill。

专业领域的 Skill，很多时候并不会轻易开源。但对于服务中小企业的公共服务平台而言，开源共创本身就是一条很有意义的路。

---

## 这是什么

优质中小企业自检助手（开源版）是一款面向中小企业成长培育场景的 AI Skill。

通过快速、简单地调用本技能，即可围绕企业材料中的关键数据、知识产权、产业链定位、主导产品表述和核心技术叙事等内容，开展全流程、多维度的系统化自检。AI 可辅助进行数据穿透验算、逻辑一致性核查和重点板块诊断，帮助企业查漏补缺，提升材料质量与自我认知的准确性。

它的核心作用，不是替代企业思考和判断，而是提供一套相对系统、可复用、可扩展的专业化自检框架，帮助企业把基础做得更扎实，把问题发现得更靠前，把提升方向看得更清楚。

## 适用对象

- 希望开展自我体检、查找短板、优化提升的中小企业

## 核心能力

- 🕵️ **关键指标预检**：自动提取营业收入、主营业务占比、营收增长率、研发费率等关键指标，帮助企业快速识别基础短板。
- 💸 **财务数据穿透式验算**：自动核对关键数据口径之间的一致性，识别常见的口径不统一、前后不自洽等问题。
- ⛓️ **跨表单逻辑扫描**：交叉比对行业代码、领域归属、主导产品、主营业务占比等关键字段的一致性。
- 📖 **重点叙事板块诊断**：围绕产业链配套、产学研合作、企业简介、主导产品名称等关键板块提供问题识别与优化建议。
- 🔬 **知识产权关联核查**：检查知识产权与主导产品、核心技术和企业叙事之间的支撑关系。
- 🏭 **产业链定位分析**：辅助识别企业在相关产业链中的位置、作用和表达逻辑。
- 📄 **双格式报告输出**：支持生成 Markdown 与 PDF 两种格式的诊断报告，便于内部审阅、沟通与归档。
- 🛡️ **自检闸门机制**：内置多项退出校验逻辑，尽量减少遗漏与敷衍输出。

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
playwright install chromium
```

### 2. 放入 Skills 目录

将 `sme-checkup/` 文件夹放入 AI Agent 工作空间的 `skills/` 目录下。

### 3. 直接调用

你可以直接对 AI 说：

- `帮我体检一下这份材料。`
- `读取这份文档，按 Skill 流程输出诊断报告。`
- `先抽取文本，再执行逻辑诊断和报告生成。`

---

## 典型使用流程

### 第一步：准备材料
上传或放入企业相关材料文件，常见格式包括：

- `.docx`
- `.doc`
- `.pdf`

### 第二步：调用 Skill
通过自然语言触发诊断任务。

### 第三步：自动执行分析
Skill 将按照预设流程自动完成：

- 文本提取与章节拆分
- 关键数值识别与年份推导
- 关键指标预检
- 财务数据穿透验算
- 知识产权关联核查
- 产业链定位分析
- 重点叙事板块诊断
- 结构化报告生成

### 第四步：输出结果
最终输出结构化体检报告，支持：

- Markdown
- PDF

## 推荐使用的平台

本项目以 `SKILL.md` 为核心入口，建议放在支持 Skills、自动化任务、本地工具调用和文件读写的平台中使用。

当前较适合本类开源 Skill 的平台包括：

- **OpenClaw**
- **钉钉悟空**
- **CoPaw**
- **腾讯 WorkBuddy**
- **腾讯 CodeBuddy**

> 除上述平台外，其他原生支持 Skills 机制的工具与平台，理论上亦可适配本项目。  
> 本项目的核心入口为 `SKILL.md`，凡支持类似技能加载、系统指令注入、工具调用及文件读写能力的平台，通常均可迁移接入。  
> 但需要注意的是，不同工具的平台能力、执行权限、上下文管理方式、文件处理链路以及所接入大模型的能力差异较大，最终输出效果可能存在明显波动。  
> 因此，对于诊断结论、规则判断和改写建议，使用者应结合具体平台特性与实际生成结果，保持必要的审慎识别，并进行人工复核与校正。

## 接入方式说明

本仓库的核心运转逻辑封装在 `SKILL.md` 中。实际接入时，通常可采用以下方式：

- 将 `SKILL.md` 作为系统提示词、技能主指令或核心工作流入口
- 将 `references/` 作为规则基准库或知识库内容
- 将 `scripts/` 作为可调用脚本工具
- 将 `reports/` 作为最终报告输出目录

在支持 Skills 的平台中，可直接以 Skill 方式调用；  
在不支持原生 Skills 的平台中，也可通过“系统提示词 + 工具调用 + 文件读写”的方式进行迁移接入。

## 项目架构

```text
sme-checkup/
├── SKILL.md                        # 技能主指令入口（核心工作流引擎）
├── README.md                       # 本说明文件
├── LICENSE                         # MIT 开源协议许可
├── requirements.txt                # Python 及渲染依赖项
├── self_review.md                  # 核心审查退出清单定义
├── reports/                        # 生成的诊断报告输出目录
├── scripts/                        # 引擎控制脚本
│   ├── extract_text.py             # 文档核心内容提取器
│   ├── self_review.py              # 自检闸门复核脚本
│   └── generate_diagnosis_pdf.py   # PDF 渲染生成脚本
├── references/                     # 规则与知识基准库（可插拔替换）
│   ├── review_criteria.json        # 指标基准层（当前：小巨人对标 v1.0）
│   ├── key_industry_chain_map.json # 产业图谱层
│   ├── expert_knowledge.md         # 知识规则层
│   └── report_template.md          # 报告模板层
└── skills/
    └── expert-dual-assessment/     # 内嵌子模块：双维度产业赛道分析引擎
```

## 可扩展架构

本技能采用**指标体系与诊断引擎分离**的可插拔架构设计，确保具备较好的扩展能力：

- 接入新指标体系 → 新建或替换 `review_criteria.json`
- 新增行业经验规则 → 在 `expert_knowledge.md` 中追加条目
- 扩展产业链覆盖 → 在 `key_industry_chain_map.json` 中增减条目
- 调整报告风格 → 修改 `report_template.md`

后续可以继续扩展更多优质中小企业培育评价体系，以及更多行业化诊断能力。

## 使用边界说明

> [!WARNING]
> 本工具定位为企业材料自查与能力提升辅助工具，不构成任何官方认定、评审结论或结果承诺。

本工具输出内容基于输入材料、规则库和算法分析生成，主要用于逻辑互查、结构诊断、风险提示和文本优化辅助。  
对于模型生成的诊断结论、修改建议和风险提示，使用者仍应结合企业真实情况、最新要求及专业人员意见进行人工判断，不宜脱离实际直接照单采用。
