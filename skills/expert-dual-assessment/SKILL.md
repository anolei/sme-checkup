# 产业定位自查辅助模块 (expert-dual-assessment)

## 概述

**产业定位双维度自查**——为专精特新企业申报书的核心产业地位提供预检与分析辅助。

在企业自查场景中，本模块帮助企业梳理两个核心问题：
1. 的主导产品是否处于国家重点产业链关键环节？
2. 企业所在领域是否属于国家重点支持的产业领域？

每维度给出**材料表达与实际匹配度综合评估**：
- **牵强挂靠（契合度弱，依赖文字包装）**：实际产品/领域不在重点之列，但申报书逻辑上可自圆其说。提示申报存在横向比较风险。
- **支撑不足（契合度优，但表达欠缺）**：实际符合重点领域，但申报材料论证不足（最为常见）。提供精准的政策补强和改写方向。

> [!IMPORTANT]
> **技术深度界定标准**：执行分析前，必须读取 `../../references/expert_knowledge.md` 中的"技术深度自查（J-05）"：
> - **技术突破特征**：自研核心算法、性能超越国际、实现国产替代、解决卡脖子问题
> - **技术应用特征**：使用第三方平台、只有名词概念、聚焦功能实现、无详细参数
> 
> 严格区分"名词包装"与"真实技术壁垒"，帮助企业准确定位。

## 使用方式

### 1. 作为子技能（自动调用）
被 `open-application-checkup` 主技能在"深入分析与优化建议"阶段自动调用。

### 2. 单独使用
```
用户：帮忙看看这家企业的产业链属于关键环节吗？
→ 读取申报书第七章、第八章、企业简介
→ 输出产业定位自查分析报告
```

## 输入

- 申报书第七章（产业链配套）
- 申报书第八章（主导产品所属领域）
- 企业简介（2000 字大作文）

## 工作流程

### ⚠️ 前置要求：必须加载参考文件

**在开始分析前，必须先读取以下文件**：

1. **读取** `../../references/key_industry_chain_map.json`

**如未加载上述文件，输出中必须标注"⚠️ 提示：未获取产业链图谱缓存，对比结果仅作基础参考"**

---

### 阶段一：信息提取
1. 读取申报书第七章（产业链配套）
2. 读取申报书第八章（主导产品所属领域）
3. 读取企业简介

### 阶段二：维度一自查（产业链环节）
1. **匹配产业链**：使用 `key_industry_chain_map.json` 寻找对应链条。
2. **定位层级**：分析位于上游基础层、中游核心层还是下游应用层。
3. **情形归类（无需打标签，直接展示结论建议）：**
   - **表达清晰且契合度高** → 优势明显，具备强竞争力
   - **契合度较好，支撑论述不足** → 提示补强国家战略价值论述
   - **存在牵强挂靠风险** → 提示横向竞争力可能不足
   - **契合度弱且表达不清** → 提示申报存在极大风险
4. **问题排查**：是否缺国际对标？是否缺"打破垄断"表述？是否处于链条末端（下游）？
5. **优化建议**：升格立意，指导向国家战略侧靠拢。

### 阶段三：维度二自查（国家重点领域）
1. **匹配领域**：将企业所属领域与重点产业/新兴产业/未来产业对照。
2. **五基判定**：核对《产业基础创新发展目录》。
3. **情形归类**（同上）。
4. **问题排查**：是否缺少引用规划文件？工业"六基"是否误填/空填？
5. **优化建议**：明确列出应挂靠的具体政策和五基名目。

### 阶段四：输出报告
1. 生成自查分析表格。
2. 提供参考改写示范（标注：AI参考改写，仅供修改思路参考）。
