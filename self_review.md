# 体检报告自我复核机制

**版本**: v2026.03-open  
**用途**: 体检报告生成后，强制执行自我复核。复核不通过时，agent 必须回到对应缺失步骤严格按照 SKILL.md 重新执行，补齐后重新生成报告，再次运行复核，直到全部通过为止。

---

## 复核流程

```
agent 按 SKILL.md 生成报告
       ↓
运行 self_review.py 检测 14 项
       ↓
全部通过？ ──→ 是 → 输出终稿（MD + PDF）
       ↓
       否 → 回到 SKILL.md 对应阶段重做缺失步骤
              ↓
         重新生成报告 → 再次运行 self_review.py
              ↓
         （循环直到通过）
```

---

| 3 | 跨表单逻辑一致性是否核查 | 阶段三 3.2 | 叁 3.1 |
| 4 | 经济数据穿透式验算是否执行 | 阶段三 3.3 | 贰 |
| 5 | 知识产权深度核查是否完成 | 阶段 2.5 | 叁 3.2 |
| 6 | 双维度产业定位评判表格是否输出 | 阶段四 4.0 | 肆 |
| 7 | 产业链配套描述诊断与改写骨架是否生成 | 阶段四 4.1 E-01 | 伍 2 |

## 执行方式

### 自动执行（脚本）

```bash
python3 scripts/self_review.py reports/[企业名称]_体检报告.md
```

脚本会逐项扫描报告内容中的关键词命中情况，输出通过/未通过结果。

### 脚本返回值

| exit code | 含义 | agent 应执行的动作 |
|:---:|--------|--------|
| 0 | 全部通过（或仅有警告） | 输出终稿 |
| 1 | 存在严重缺失 | **回到 SKILL.md 对应阶段重新执行缺失步骤，重新生成报告，再次复核** |

---

## 复核结果输出格式

复核完成后，脚本会自动输出以下格式的报告：

```
============================================================
申报书体检报告自我复核报告
============================================================

🔴 发现 X 个严重缺失（必须重做）：
  ✗ [序号] 具体缺失描述

🟡 发现 X 个警告（建议完善）：
  △ [序号] 具体警告描述

============================================================
❌ 复核未通过
→ agent 必须严格按照 SKILL.md 重新执行上述缺失步骤，
  补齐后重新生成报告并再次执行本脚本复核。
```

## 触发条件

| 触发场景 | 触发方式 |
|---------|---------|
| 体检报告生成后 | **自动触发**（阶段六强制执行） |
| 用户质疑体检结果不完整 | 手动重新运行 |
| 报告补充修正后 | 再次运行验证 |
